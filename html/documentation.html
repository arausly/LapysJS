Do not use with other components

Icon classes are made via CSS and not HTML/ Javascript.
In that case, it would be safer to apply them separately from all other elements or components to prevent disarray in any styling.

Only for empty elements

Icons must be in use for childless or non-text containing elements.

Icons are not as they seem

When an icon is copied, it is not the vector itself that is sent to the clipboard but rather the letter in the font that is taken to temporary storage.
Icons should also be treated as regular text in HTML markup structure.
Also take note that Lapys JS icons are not conventional but still effective in today’s modern web.

Not screen-readable

Icons, being made from CSS alone are not screen-readable and are meant for decorative uses only.

Buttons & Components (Widgets)

Default and bare-bone styled button widgets (elements meant for functionality).
They become buttons via the .btn (button) class
and widgets through the .cmpnt (component) class.
Though it may not seem like it initially, button components have an unlimited range of application and use in design and style as per their tolerance for grids and non-jarring, appealing and simple look.

Use a .btn and .cmpnt class on any element.

    Bare Styling

    Accordions, just as most other components are barely styled at all on their own. This is to prevent any problems or stress to the developer in customizing the accordion to fit a particular design.
    Although preset accordion styles still exist and widgets can be used to beautify the markup of the components.

    For more on styling accordions, click here…

    Opening and Closing Accordions

    Accordions are by default closed, but if you want to toggle them open, append a data-open attribute on the accordion container.

    Dropdowns

    Panels that hold content of data within a separate container.

    Apply a class attribute with a value of drpdwn to an element for the dropdown header.
    For the content, reference the header with a data-drpdwn attribute to the header’s id.
    There are various ways of toggling a dropdown's menu or content (clicking, with the cursor… and so on).
    That is why dropdowns require the data-event attribute to effect how the component is interacted with.
    This following a list of events and the corresponding data-event attribute as such that could be utilized for dropdowns.
    • data-event="on" for clicking.
    • data-event="over" for hovering.
    • data-event="over_toggle" for exiting and hovering.

    Lapys JS Dropdowns are not Drop Menus

    Even though they are called and coded as dropdowns, Lapys JS dropdowns act more as drop menus because they have more flexibility and are not fixed to a specific mark-up.

    Lapys JS Dropdowns are definitely not Accordions

    Dropdowns perform similar functions to accordions but the difference between a dropdown and an accordion is that accordions keep both the content and header together within a container or parent element.

    Navigation Panel

    Similar to a button group but rather than the extra cmpnt-grp (component group) class
    use the nav class.
    Another difference is that its children do not need any classes to be identified as navigation items.

    Apply a class of nav to an element (component) with a cmpnt class.
    Different classes can be applied to a navigation panel with each style affecting the style of the navigation and its items.
    The following listed are how to get these styles:
    • An attribute of psd-focus on a panel item will style it to seem focused.
    • A class of fill on the container element.
      Spans the items to fill up the panel.
    • A class of tab on the container element.
      Styles the panel items to represent tabs.
