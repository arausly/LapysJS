Not for Screen Readers

All LapysJS icons are made for decorative uses only.
Being made from fonts as well, a screen-reader would pronounce the character that makes the icon.

Do not use with other components

Icon classes are made via CSS and not HTML/ Javascript.
In that case, it would be safer to apply them separately from all other elements or components to prevent disarray in any styling.

Only for empty elements

Icons must be in use for childless or non-text containing elements.

Icons are not as they seem

When an icon is copied, it is not the vector itself that is sent to the clipboard but rather the letter in the font that is taken to temporary storage.
Icons should also be treated as regular text in HTML markup structure.
Also take note that Lapys JS icons are not conventional but still effective in todayâ€™s modern web.

Frames

Scalable elements meant for embedded objects (thumbnail) and quick captions.

The first and last <img> elements within the .frm class elements are thumbnail images
while all other child elements are considered captions.
Even though frames were meant to hold captions and thumbnails, they can still function otherwise but in general are meant to hold elements within cards.

Responsive Embeds

Automated sized element that is fully compatible with grids and smaller screen sizes.

Press on a rspnv class on a block-level based element (such as <img>’s, <table>’s, <video>’s and so on).

Specific Responsive Embed

To specify the kind of responsive content, apply the element’s tag name as a modifier class and then the base class e.g:

  • .rspnv-embed for <embed> elements.
  • .rspnv-img for <img> elements.
  • .rspnv-table for <table> elements.

Responsive Children

To make all embedded content within a parent element responsive, append a class of .rspnv to the parent.

Un-responsive Children

To remove responsiveness on an element, append the .no-rspnv class value on the element.

Toggling Accordions

Accordions are by default closed, but if you want to toggle them open, append a data-open attribute on the accordion container.

Bare-Boned Styling

Accordions, just as most other components are barely styled at all on their own.
This is to prevent any problems or stress to the developer in customizing the accordion to fit a particular design.
Although preset accordion styles still exist and widgets can be used to beautify the markup of the component.

Dropdowns

Panels that hold data within a separate container.

Apply a class attribute with a value of .drpdwn to an element for the dropdown header.
For the content, reference the header with a data-drpdwn attribute to the header’s id.
There are various ways of toggling a dropdown’s content (or menu) (clicking, with the cursorâ€¦ and so on).
That is why dropdowns require the data-event attribute to effect how the component is interacted with.
The following a list of events and the corresponding data-event attribute as such that could be utilized for dropdowns.

  • data-event="on" for clicking.
  • data-event="over" for hovering.
  • data-event="over_toggle" for exiting and hovering.

LapysJS Dropdowns are not Drop Menus

Even though they are called and coded as dropdowns, Lapys JS dropdowns act more as drop menus because they have more flexibility and are not fixed to a specific mark-up.

LapysJS Dropdowns are definitely not Accordions

Dropdowns perform similar functions to accordions but the difference between a dropdown and an accordion is that accordions keep both the content and header together within a container or parent element.

Navigation Panel

Similar to a button group but rather than the extra .cmpnt-grp (component group) class use the .nav class. Another difference is that its children do not need any classes to be identified as navigation items.

Apply a class of .nav to an element (component) with a .cmpnt class. Different classes can be applied to a navigation panel with each style affecting the style of the navigation and its items.
The following listed are how to get these styles:

  • A class of fill on the container element spans the items to fill up the panel.
  • A class of pill on the container element styles the panel items to represent pills.
  • A class of stack on the container element styles the panel items to represent tabs.
  • A class of tab on the container element styles the panel items to represent tabs.
  • An attribute of psd-focus on a panel item will style it to seem focused.

Message Boxes

Contextual placeholders for messages informing the user of key things that must be taken in consideration.

Message boxes do not have a base class but have a modifier class.
Basically, utilize a class of msg- on an element, then modify it with any of the following listed below:

  • msg-description allows for descriptive messages.
  • msg-error for error messages.
  • msg-info for messages meant to inform.
  • msg-valid for success or valid messages.
  • msg-warn for messages that warn the reader.
You could also Edit Code to test how each modifier affects the base.
