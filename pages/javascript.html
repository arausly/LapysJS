LapysJS Javascript

Bring LapysJS’ components to life with already built-in or selected plug-ins.

Overview

Imbed

LapysJS javascript (*.js) files can be included all-in-one or one-by-one.
In the sense that, the lapys.js or lapys.min.js files include all plug-ins of LapysJS while
other javascript files include only a portion of the whole.

All for One, and One for All

It is better considered to embed only the specific LapysJS plug-ins needed for a project.

data-* Attributes

LapysJS javascript inputs data-* attributes only to modify HTML elements.

Variables

LapysJS was built with a few global variables already accessible.
All of which are to simplify coding time & development.

lapys

The lapys variable collects the element using a Javascript function the moment the function is called.
For example, if a variable assigned to a getElementsByTagName() function is used, lapys stores the variable’s value (which is the function).
However, there a few exceptions to the rules of using the lapys variable.
For more information. you can check the guide page.

Typically when a function is used, lapys updates itself.

LapysJS Object

The LapysJS object embedded within the JS file gives a JSON-level of information about the framework.
Basic information about LapysJS makes up the object’s properties.
Information such as the file’s embed element (<script>), the execution date & time, the version of the framework and so on.

create()

LapysJS creates a host of shortened Javascript functions to simplify and prevent the stress in writing code.
create() is the first of these abbreviated functions.

The create() method creates the HTML element specified by its tagName,
or an HTMLUnknownElement if the tagName is not recognized.
In an XUL document, it creates the specified XUL element.
In other documents, it creates an element with a null namespace URI.

create("p") // Creates a <p> element.

css Object

LapysJS’ css object allows for a variety of function methods
relating to CSS and style on the DOM through its properties to be used for dynamic web pages.

css.add()

The add method allows for <link> elements to
be appended dynamically within the DOM.
This is useful in changing the theme or styles of a page dynamically, instantly or interactively.

Javascript
/* The properties of the link must be added into the function as well. */ css.add("lapys.css", "screen", "stylesheet", "text/css")
HTML
<link href="lapys.css" media="screen" rel="stylesheet" type="text/css">
css.style()

The style method is used to append <style> elements
the same way <link>’s are added via the css.add() method.

Javascript
/* The _q_data-key_q_ parameter is used to identify the <style> element */ css.style( /* <style> data-key Attribute */ "customStyle", /* CSS Selectors */ "body, html", /* CSS Properties */ "background-color: #F7F7F7; " + "font-size: 16.6px;", /* CSS At-Rules (not necessary) */ "@keyframes fadeIn {" + "0% {" + "opacity: 0" + "}" + "}" )
HTML
<style data-key="customStyle"> body, html { background-color: #F7F7F7; font-size: 16.6px; } @keyframes fadeIn { 0% { opacity: 0 } } </style>

del Object

The LapysJS del object much like the css object
allows the developer to remove key things in the document, such as:
dynamically added styles, element classes, event listeners, inline-styling and so on…

del.attr()

The attr method under the del object removes attribute nodes from an element.
Simple as that.

Javascript
del.attr(document.getElementById("myParagraph"), "style")
HTML
<p id="myParagraph" > </p>
del.class()

The class method removes a class node from an element.

Javascript
del.class(document.getElementById("myParagraph"), "classA")
HTML
<p class=" classB" id="myParagraph"> </p>
del.event()

The event method removes from the EventTarget an event listener
previously registered with an EventTarget.addEventListener() method.
The event listener to be removed is identified using a combination of the event type,
the event listener function itself and other functions.

Javascript
del.event(element, event, function)
del.link()

The link method removes a specified <link> element from the DOM.

Javascript
del.link("lapys.css", "screen", "stylesheet", "text/css")
HTML
del.html()

The html method removes a specified element from the DOM.

Javascript
/* The element is queried for, then found with the numeric index: del.html(element, index). */ del.html(document.getElementById("myParagraph"), 0)
HTML
del.inlineStyle()

The inlineStyle method removes a specified inline style from an element in the DOM.

Javascript
/* del.inlineStyle(element, style) */ del.inlineStyle(document.getElementById("myParagraph"), "height")
HTML
<p id="myParagraph" style="font-size: inherit; ;"> </p>
Removed Styling

If the style attribute of an element is null, it is removed by the inlineStyle function.

del.style()

The style method removes a specified <style> element from the DOM.

Javascript
/* The <style> element to be removed is accessed from it_q_s _q_data-key_q_ attribute. */ del.style("customStyle")
HTML

dir Object

The LapysJS dir object allows the developer
to access specified parts of the document’s URL.

dir.fullPath

This property returns the entire URL (of the current page).

dir.origin

This property returns the protocol, hostname and port number of the current page’s URL.

dir.path

This property returns the path name of the current URL.

file Object

The LapysJS file object allows the developer
to access, delete, edit, read and write files on the local disk.

Experimental Technology

The LapysJS file object is still being developed on
to fit standards and concur with modern Javascript.

file.close

This property allows the developer to close initially opened files.

file.close(directory, type)
file.contentType

This property returns the MIME type that the document is being rendered as.
This may come from HTTP headers or other sources of MIME information,
and might be affected by automatic type conversions performed by either the browser or extensions.

Non-Standard

This feature is non-standard and is not on a standards track.
Do not use it on production sites facing the Web: it will not work for every user.
There may also be large incompatibilities between implementations and the behavior may change in the future.

file.domain

Gets/ sets the domain portion of the origin of the current document,
as used by the same origin policy .

file.inputEncoding

This property returns the character encoding for the document.
The character encoding is the character set used for rendering the document,
which may differ from the encoding specified by the page (the user can override the encoding).
It returns the character encoding at the time of parsing,
and will return null if the document is created in memory.

document.characterSet

This property is an alias of document.characterSet.

file.lastModified

This property returns a string containing the date and time on which the current document was last modified.

file.name

This property returns the document’s file name.

file.open

This property allows the developer to open closed files.

file.open(directory, type)
file.plugIns

This property returns an HTMLCollection object containing one or more
HTMLEmbedElements or null which represent the <embed> elements in the current document.

NavigatorPlugins.plugins

One can also refer to the NavigatorPlugins.plugins property which
returns a PluginArray object, listing the Plugin objects describing the plugins installed in the application.

NavigatorPlugins.plugins (continued)

Although in Firefox 29 and later, enumeration of the navigator.plugins array may be restricted as a privacy measure.
Applications that must check for the presence of a browser plugin should query navigator.plugins
or navigator.mimeTypes by exact name instead of enumerating the navigator.plugins
array and comparing every plugin's name.
This privacy change does not disable any plugins; it just hides some plugin names from enumeration.

file.read

This property allows the developer to read files on the local disk.

file.read(directory, type)
file.ready

The ready property describes the loading state of the document.

file.scripts

This property returns a list of the <script> elements in the document.
The returned object is an HTMLCollection.

file.stylesheets

This read-only property returns a StyleSheetList of CSSStyleSheet objects for stylesheets explicitly linked into or embedded in a document.

file.type

This property returns the document’s file type.

file.visibility

This read-only property returns the visibility of the document,
that is in which context this element is now visible.
It is useful to know if the document is in the background or an invisible tab, or only loaded for pre-rendering.

file.write

This property allows the developer to write files on the local disk.

file.write(content, directory, type)
file.XML

This object property allows the developer to read all information pertaining to XML on the document.

file.xmlEncoding

This property returns the encoding as determined by the XML declaration.
Should be null if unspecified or unknown.

LapysJS’ get object allows the developer to select
specified data from the DOM or elements on the DOM.

HTML

lorem ipsum ...

The get.class() method is only used as a shortcut or second option
for the get.attr() which can also inquire the same function as the get.class() method.

The css method gives the specified value of the CSS property of an element after applying the active stylesheets and resolving any basic computation those values may contain.

HTML

lorem ipsum ...

get.html()

The html method returns a list of the elements within the document
(using depth-first pre-order traversal of the document’s nodes) that match the specified group of selectors.
The object returned is a NodeList.

HTML

lorem ipsum ...

Javascript
/* get.html(selector, index) */ get.html(document.getElementById("myParagraph"), 0) // returns _q_p#myParagraph_q_.
Selectors

The get.html() method selects elements via their CSS selector.

Indexing

The get.html() method does not need an index to access the element if only one instance of the element specified exists.

Arrays

If an array or NodeList is required, the index variable be replaced with the "_array" string e.g.:
get.html(document.getElementById("myParagraph"), "_array").

js Object

LapysJS’ js object allows for a variety of function methods
relating to Javascript embeds on the DOM through its properties to be used for dynamic web pages.

js.add()

The add method allows for <script> elements to
be appended dynamically within the DOM.
This is useful in changing the functionality of a page dynamically, instantly or interactively.

Javascript
/* The properties of the script must be added into the function as well. */ js.add("lapys.js", "text/javascript", "async")
HTML
<script async="" src="lapys.js" type="text/javascript"> </script>
js.script()

The script method is used to append <script> elements into the DOM with some parsed code.

Javascript
/* The _q_data-key_q_ parameter is used to identify the <script> element */ js.script( /* <script> data-key Attribute */ "customScript", /* Simple Code */ "function() {" + "// Do something..." + "}" )
HTML
<script data-key="customScript"> function() { // Do something... } </script>

log()

This function allows the developer to print data within the browser’s console.

log("Some data.") // prints _q_Some data._q_
Source Finding

Though this may be a shorter version of logging data into the console
finding the line of code where it was executed will always be fixed to a point in the LapysJS script file
and not the file that used the function method.

parseBool()

This function returns either true or false for any data type depending on its primitive boolean value.

parseBool("Some data.") // returns _q_true_q_.
